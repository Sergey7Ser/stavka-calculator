# Калькулятор-услуга под ключ

Продукт, который мы делаем повторяемо: **прайс клиента → одностраничный фирменный
калькулятор → смета → коммерческое предложение (PDF) → публичная ссылка на поддомене**.
Эталонная реализация — `astralium.html` в этом репозитории. Новый калькулятор делай
по её образцу, не изобретай архитектуру заново.

## Процесс (проверен на ASTRALIUM, июль 2026)

### 1. Разбор прайса

- Excel читай питоном: `pip3 install openpyxl`, дампи все листы с буквами колонок.
- Ищи **двухуровневые цены** (розница / нижняя граница) — в наших прайсах это норма.
  В калькуляторе они становятся переключателем «Стандарт / Нижняя граница».
- Чего в прайсе обычно НЕТ: цен на материал. Ставь ориентировочные рыночные,
  помечай «правится», поле редактируемое.
- Коэффициенты сложности (×1.15…×1.3) — отдельная таблица; применяются к работам,
  НЕ к материалу и не к «прочим услугам» (замер/доставка). Перемножаются.

### 2. Архитектура HTML (копируй из astralium.html)

Один самодостаточный файл, без сборки и CDN. Скелет:

- `const DB = { разделы: [{id, name, unit, std, min}], EDGES-матрицы, COEFS, STONES }` —
  весь прайс данными, строки рендерятся JS (createElement + textContent, не innerHTML).
- Состояние `S` (mode, qty, priceOv, строки материала, коэффициенты, kp) →
  `localStorage`, ключ вида `<бренд>_calc_v1`. `load()` мержит с дефолтами.
- Каждая цена — редактируемый `<input>` прямо в строке; правка пишется в
  `S.priceOv[id][mode]`, подсвечивается акцентным цветом, кнопка «Сбросить цены к прайсу».
- Разделы — `<details class="card">` с бейджем суммы; длинные списки — с поиском-фильтром.
- Липкий низ (dock): «Итого» + золотая кнопка «Создать КП →».
- Расчёт: `calc()` возвращает объект с разбивкой; `recalc()` обновляет строки, бейджи,
  смету. Числа — `Intl.NumberFormat('ru-RU')`, в таблицах `tabular-nums`.

### 3. Страница КП (второй экран в том же файле)

- `body.kp-open` скрывает калькулятор, показывает белый «лист» (`.kp-sheet`) на тёмном фоне.
- Лист: шапка с брендом и № КП (генерится из даты `ГГММДД-1`, редактируется) + «действительно
  до +14 дней»; поля Кому/Объект; смета таблицей с разделами; крупный итог; комментарий;
  5–7 преимуществ **через выгоды клиента**; условия; контакты (телефон/почта/менеджер —
  сохраняются в localStorage навсегда).
- PDF = `window.print()` + print-CSS: печатается только лист, плейсхолдеры прозрачные
  (`::placeholder {color: transparent}`), пустой комментарий скрыт
  (`.kp-comment:has(textarea:placeholder-shown) {display:none}`), `break-inside: avoid`.
  Имя файла = `document.title` — ставь «КП <БРЕНД> № …» при открытии КП.
- «Поделиться» — `navigator.share`, фолбэк в буфер обмена с тостом.

### 4. Дизайн

- Не копируй чужой сайт вслепую (обычно закрыт от ботов, 403). Палитру строй из ниши:
  камень = графит #101215 + латунь #C9A96E; «Опора Бизнеса» = синий #142634 + золото #D9BF85.
- Серифный логотип с letter-spacing, тёмная премиальная тема для B2B-ниш; лист КП — всегда
  светлый (это документ).
- Мобильный вид обязателен: владелец считает с телефона. Сетки сужай в @media 720px.

### 5. Проверка (перед каждым пушем)

Headless-прогон: заполни поля, сверь итог с ручным расчётом, переключи режим цен,
перезагрузи (localStorage), открой КП, сними скриншоты экрана и print-эмуляции:

```js
import { chromium } from 'file:///opt/node22/lib/node_modules/playwright/index.mjs';
const b = await chromium.launch({ executablePath: '/opt/pw-browsers/chromium-1194/chrome-linux/chrome' });
// p.emulateMedia({media:'print'}) → скриншот печатной версии; p.pdf() → проверка A4
```

Playwright в проект не ставить (репозиторий — только HTML + brand/).

### 6. Публикация

Конвейер уже настроен: **пуш в `main` → `.github/workflows/pages.yml` → GitHub Pages**.

- Рабочая ветка → `git merge --ff-only` в `main` → push. Через ~2 мин сайт обновлён.
- Новый поддомен: владелец в reg.ru («Домены» → домен → «Управление зоной DNS» →
  «Добавить запись») создаёт `CNAME <имя> → sergey7ser.github.io`, затем в
  GitHub Settings → Pages вписывает Custom domain и после зелёной галочки ставит
  «Принудительно HTTPS (Enforce HTTPS)».
- Один Pages-сайт = один custom domain. Несколько калькуляторов на разных поддоменах —
  через переадресацию по hostname в `index.html`:
  `if (location.hostname === 'kamen.oporabusiness.ru') location.replace('/astralium.html')`.
- Артефакт-превью для владельца: копия файла без `<!DOCTYPE>/<html>/<head>/<body>`
  (обёртку даёт платформа), init через проверку `document.readyState`, публикуй тем же
  файлом → URL сохраняется.

## Грабли (наступали — не повторяй)

1. **Включить Pages может только владелец репозитория** — робот получает
   «Resource not accessible by integration». Проси один клик: Settings → Pages →
   Source: «GitHub Actions».
2. Первый деплой после включения Pages часто падает «Deployment failed, try again later» —
   это временное; перезапуск = пустой коммит в `main` (API rerun тоже закрыт для робота).
3. **Песочница без сети**: все внешние домены → 403 от прокси, SSH наружу закрыт.
   Живые URL проверяет владелец глазами; статус деплоя смотри через GitHub MCP
   (`actions_list` / `get_job_logs`).
4. Владелец пользуется телефоном и авто-переводом браузера: кнопки называй по-русски
   с английским оригиналом в скобках, шаги — короткие, по одному действию, со ссылками.
5. `npm install` в проекте оставляет package.json/node_modules — удаляй, дерево держи чистым
   (стоп-хук требует отсутствия неотслеживаемых файлов).
6. Публичная ссылка показывает ВЕСЬ прайс, включая «нижнюю границу». Перед отправкой
   клиентам предложи владельцу клиентскую версию без внутренних цен.

## Бэклог идей (владелец упоминал)

- Клиентская версия без «нижней границы» и внутренних пометок.
- Логотип Astralium в шапке и на листе КП.
- Форма заявки («Хочу расчёт») + аналитика посещений.
- Новые ниши: керамогранит отдельно, ремонт, другие услуги — тот же рецепт.
